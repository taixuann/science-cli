## Objective
Integrate the premium frontend dashboard generated by Google AI Studio into `src/science_cli/serve/frontend` while upgrading the Python web server (`server.py` and `api.py`) to fully support dynamic project discovery and project-level routing context without breaking any existing CLI or data features.

## Context & Key Findings
During integration, three critical issues were identified that caused a blank page or file loading errors:
1. **Missing `get_protocol_files` Function**: The `/api/protocol/<name>/files` route in `server.py` calls `api.get_protocol_files()`, but this function was completely missing in `src/science_cli/serve/api.py`. This caused requests from the frontend to return a `500 Internal Server Error`, showing the message "Failed to load protocol files: 2_phd-investigation".
2. **Missing `_serve_project_file` Handler**: The `/files/<path>` route in `server.py` delegates to `self._serve_project_file()`, but the method was never defined on `SciServeHandler`. This caused visual assets (plots, SVGs, PDFs) to fail to load.
3. **Step Schema Mismatch (React Crash)**: The React dashboard (`App.tsx`) expects `protocols[].steps` to be a list of `StepItem` objects (dicts with `name` and `files` array). However, the real backend returned `steps` as a simple list of strings (`["1_initial", "2_sweep"]`). When React tried to render `activeStep.files`, it crashed with `Cannot read properties of undefined (reading 'map')`, resulting in a completely blank page.

## Proposed Changes

### 1. Server-Side APIs & Project Resolution (`src/science_cli/serve/api.py`)
- **Modify `_scan_protocol_dirs`**: Return `steps` as a list of dicts with `name` and `files` keys to match the `StepItem` schema.
- **Add `_get_step_files` Helper**: Scan step subdirectories (both the directory itself and the `results/` folder) for `.png`, `.pdf`, and `.svg` files, returning structured file metadata (name, path, type, size, created).
- **Implement `get_protocol_files`**: Scan a specific protocol's steps and return all step and file structures in the correct format.

### 2. Static File Serving (`src/science_cli/serve/server.py`)
- **Implement `_serve_project_file`**: Define the missing handler on `SciServeHandler`. Resolve files safely relative to `get_projects_root()`, verify that the path remains within the projects root directory (preventing directory traversal attacks), and serve them with correct MIME types and headers.
